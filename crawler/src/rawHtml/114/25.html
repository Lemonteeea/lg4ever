今天开始进行算法模板的复习和整理。授人以鱼，不如授人以渔。在本讲，我的目的是教会你如何做知识的整理和模板的整理，而不是直接给你一些现成的东西，让你去死记硬背。无论是思维导图，还是代码模板，你自己整理一遍的收获会更大。

今天我们主要介绍两种方法：

  • 通过思维导图将学过的知识添加到你的知识树中；

  • 将刷过的题目整理成代码模板，放到你的代码模板库中。

排序

我们在学习排序的时候，主要讨论了合并模板和快速排序两种排序，现在就可以利用下面这个思维导图进行快速复习。

合并的技巧

对于合并排序来说，我觉得最重要的是掌握下面这段合并的小技巧：

复制代码 
  1.     int i = b;

  2.     int j = m;

  3.     int to = b;

  4.     

  5.     // 将两个子数组进行合并, 注意下面是一个很重要的模板

  6.     // 这里的判断条，是只要两个子数组中还有元素

  7.     while (i < m || j < e) {

  8.       // 如果右子数组没有元素 或 左子数组开头的元素小于右子数组开头的元素

  9.       // 那么取走左子数组开头的元素

  10.       // 考点：a[i] <= a[j]这样可以保证合并排序是稳定的，不要写错!

  11.       if (j >= e || i < m && a[i] <= a[j]) {

  12.         t[to++] = a[i++];

  13.       } else {

  14.         // 否则就是取右子数组开头的元素

  15.         t[to++] = a[j++];

  16.       }

  17.     }

代码：Java/C++/Python

可以看到 while 语句中 if 语句的写法（上述代码第 11 行）用了最简洁的代码处理了各种边界条件！

三路切分

在《08 | 排序：如何利用合并与快排的小技巧，解决算法难题？》介绍快速排序时，三路切分也是一个高频出现的知识点，所以我们需要掌握这个代码模板，如下所示：

复制代码 
  1. // 辅助函数：交换数组中的两个元素

  2. void swap(int[] A, int i, int j) {

  3.   int t = A[i];

  4.   A[i] = A[j];

  5.   A[j] = t;

  6. }

  7. // 三路切分

  8. int threeSplit(int[] A, int b, int e) {

  9.   if (b >= e) {

  10.     return 0;

  11.   }

  12.   // 我们取数组中间的数 

  13.   final int m = b + ((e - b) >> 1);

  14.   final int x = A[m];

  15.   // 注意我们的初始区间有四个：

  16.   // [b, l) [l, i) [i, r] (r, N)

  17.   // [小于)  [等于) [未知]  (大于)

  18.   int l = b;

  19.   int i = b;

  20.   int r = e - 1;

  21.   while (i <= r) {

  22.     if (A[i] < x) {

  23.       swap(A, l++, i++);

  24.     } else if (A[i] == x) {

  25.       i++;

  26.     } else {

  27.       swap(A, r--, i);

  28.     }

  29.   }

  30.   // 切分完毕之后，只有三个区间

  31.   // [b, l) [l, i) [i, N)

  32.   // 首先看中间的区间

  33.   if (i - l == 1)

  34.     return A[l];

  35.   // 再看左区间

  36.   if (((l - b) & 0x01) == 1) {

  37.     return threeSplit(A, b, l);

  38.   }

  39.   // 再看右区间

  40.   return threeSplit(A, i, e);

  41. }

代码：Java/C++/Python

二分

我们将二分的知识点整理如下：

当已知一个题目可以用二分进行破解时，就可以使用 lowerBound 和 upperBound 这两个标准的二分的模板了。

此外，我们还需要记住这两个模板的使用的条件：

  • lowerBound 用于查找有序数组中最左边出现的元素（闭）。

  • upperBound 用于查找有序数组中最右边出现的元素（开）。

注意，我们在条件部分加了“开闭”两个字！其含义如下：

当给定数组 A[] = {1,2,2,2,3}，运行 lowerBound(A, 2) 返回下标 1，而运行 upperBound(A,2) 却返回下标 4，但此时 A[4] = 3。

因此，我们还需要注意，lowerBound 与 upperBound 返回值组成的区间是一个左闭右开的区间，如下所示：

复制代码 
  1. [lowerBound(A,2), upperBound(A,2)) = [1, 4)

一定要记住这里的左闭右开原则！

lowerBound

其中 lowerBound 的代码模板如下：

复制代码 
  1. int lowerBound(long[] A, int n, long target) {

  2.   int l = 0, r = n;

  3.   while (l < r) {

  4.     final int m = l + ((r - l) >> 1);

  5.     if (A[m] < target) {

  6.       l = m + 1;

  7.     } else {

  8.       r = m;

  9.     }

  10.   }

  11.   return l;

  12. }

代码：Java/C++/Python

upperBound

upperBound 的模板代码如下：

复制代码 
  1. int upperBound(long[] A, int n, long target) {

  2.   int l = 0, r = n;

  3.   while (l < r) {

  4.     final int m = l + ((r - l) >> 1);

  5.     if (A[m] <= target) {

  6.       l = m + 1;

  7.     } else {

  8.       r = m;

  9.     }

  10.   }

  11.   return l;

  12. }

代码：Java/C++/Python

其实我们只需要记忆 lowerBound 模板就够了，因为两个模板之间的差异只有一处：

lowerBound：A[m] < target
upperBound：A[m] <= target

这里和你分享一个我的记忆方法。当 A[m] <= target 的时候，标志着 m 还可以往右移动一段距离，因为 upperBound 找到的一般都在更右边的位置，因此带有“A[m] <= target”的是 upperBound。

双指针

双指针的知识我们总结如下：

需要熟练掌握的代码模板，一共有 3 个。

最长区间

求最长区间的代码模板长成这样：

复制代码 
  1. int maxLength(int[] A) {

  2.   int N = A.length;

  3.   // 区间的左指针

  4.   int left = -1;

  5.   int ans = 0;

  6.   for (int i = 0; i < N; i++) {

  7.     // assert 在加入A[i]之前，(left, i-1]是一个合法有效的区间

  8.     // step 1. 直接将A[i]加到区间中，形成(left, i]

  9.     // step 2. 将A[i]加入之后，惰性原则

  10.     while (check((left, i]))/*TODO 检查区间状态是否满足条件*/) {

  11.       ++left; // 如果不满足条件，移动左指针

  12.       // TODO 修改区间的状态

  13.     }

  14.     // assert 此时(left, i]必然满足条件

  15.     ans = max(ans, i - left);

  16.   }

  17.   return ans; // 返回最优解

  18. }

注意：在解题的时候，需要根据具体的题目条件，在模板的基础上写一点状态更新的代码，也就是要替换掉代码模板中的“TODO”部分。

最短区间

最短区间的代码模板如下：

复制代码 
  1. int minimalRange(int[] A) {

  2.   final int N = A == null ? 0 : A.length;

  12.       ans = Math.min(ans, i - left);

贪心

虽然说贪心算法是一种思想，不过还是有一些代码模板需要你掌握。比如区间不重复的最大数目模板，如下所示：

复制代码 
  1. int nonOverlapIntervals(int[][] A) {

  2.   final int N = A == null ? 0 : A.length;

  3.   // 将区间进行排序

  4.   Arrays.sort(A, new Comparator<int[]>() {

  5.     public int compare(int[] a, int[] b) {

  6.       return a[1] == b[1] ? 0 : (a[1] < b[1] ? -1 : 1);

  7.     }

  8.   });

  9.   // 已重叠的区间的最右端点

  10.   int maxEnd = Integer.MIN_VALUE;

  11.   // 不重叠 的区间的个数

  12.   int ans = 0;

  13.   // 开始贪心算法

  14.   for (int i = 0; i < N; i++) {

  15.     final int start = A[i][0];

  16.     if (maxEnd <= start) {

  17.       maxEnd = A[i][1];

  18.       ans++;

  19.     }

  20.   }

  21.   return ans;

  22. }

代码：Java/C++/Python

此外，在《11 | 贪心：这种思想，没有模板，如何才能掌握它？》中我们介绍过“青蛙跳”问题， 实际上该解法还可以解决一系列题目，比如“第 11 讲”的练习题 5，练习题 6 等。因此，我把这个算法模板叫作青蛙跳模板。

复制代码 
  1. boolean canJump(int[] A) {

  2.   final int N = A == null ? 0 : A.length;

  3.   // 一开始，在正式开始第一次扫描之前，肯定什么元素都还没有扫描过

  4.   // 所以之记录之前扫描位置设置为-1

  5.   int preScanedPos = -1;

  6.   // 根据题意

  7.   // 当前能覆盖到数组的第0个元素。也就是当前可以够得着的元素

  8.   int curCoveredRange = 0;

  9.   // 如果当前

  10.   while (curCoveredRange < N - 1) {

  11.     int oldCoveredRange = curCoveredRange;

  12.     // 根据优化1和优化2，我们只需要遍历

  13.     // [preScanedPos + 1, oldCoveredRange]即可。

  14.     // 然后不停更新curCoveredRange

  15.     for (int i = preScanedPos + 1; i <= oldCoveredRange; i++) {

  16.       // 1. 这个区间和我们已经覆盖的范围是相连的!

  17.       // 满足相连性

  18.       // 2. 如果这个区间能覆盖得更远

  19.       if (i + A[i] > curCoveredRange) {

  20.         // 更新我们能cover的范围

  21.         curCoveredRange = i + A[i];

  22.       }

  23.     }

  24.     // 如果发现不能更新覆盖范围，说明已经没有变长的可能性了。

  25.     if (oldCoveredRange == curCoveredRange) {

  26.       return false;

  27.     }

  28.     // 我们记住上次已经扫描过的位置

  29.     preScanedPos = oldCoveredRange;

  30.   }

  31.   return true;

  32. }

代码：Java/C++/Python

回溯

关于回溯，我经常从两个方向思考：

  • 只看第 i 个人怎么选；

  • “有借有还”。

关于这两点，希望你看了下面的这个思维导图还能够想起来。

回溯的代码模板只有一个，如下所示：

复制代码 
  1. void backtrace(int[] A,

  2.                int i, /*第i个人*/

  3.                Box s, /*箱子*/

  4.                answer/*存放所有的答案*/) {

  5.   final int N = A == null ? 0 : A.length;

  6.   if (状态满足要求) {

  7.     answer.add(s);

  8.   }

  9.  

  10.   if ([i, ...., 后面）的人都没有任何选项了) {

  11.     return;

  12.   }

  13.   for 宝石 in {第i个人当前所有宝石选项} {

  14.     s.push(宝石);

  15.     backtrace(A, i + 1, s, answer);

  16.     s.pop();

  17.   }

  18. }

DFS 与 BFS

DFS 与 BFS 的知识点我压缩在了一张思维导图里：

关于代码模板，你只需要掌握两个。

DFS

通常而言，DFS 算法会用到两个模板，一个用于遍历，另一个用于收集符合条件的解。其中用于遍历的代码模板如下：

复制代码 
  1. boolean vis[N];

  2. void DFS(int start) {

  3.   if start == end {

  4.       success = true

  5.       return

  6.   }

  7.   // 遍历当前可以做出的选择

  8.   for opt in getOptions(start) {

  9.       if (vis[opt]) continue;

  10.       vis[opt] = true;

  11.       dfs(opt);

  12.       if success {

  13.           return;

  14.       }

  15.   }

  16. }

用于收集符合条件的解的代码模板如下：

复制代码 
  1. void dfs(A,

  2.          int start, /* start 表示出发点*/

  3.          vis,  /* 记录每个点是否已经访问 */

  4.          path, /* 路径*/

  5.          answer/*存放最优的答案*/) {

  6.   final int N = A == null ? 0 : A.length;

  7.  

  8.   if (状态满足要求) { // 是更好的解吗？

  9.     if (s better_than ans) {

  10.         ans = s

  11.     }

  12.     return;

  13.   }

  14.   

  15.   for next in {start点的后继点} {

  16.     if !vis[next] {

  17.       path.append(next);

  18.       vis[next] = true;

  19.       dfs(A, next, vis, path, answer);

  20.       path.pop();

  21.       vis[next] = false;

  22.     }

  23.   }

  24. }

BFS

巧合的是，BFS 也有两种写法，一种是使用队列，另一种是使用“两段击”。其中使用队列的代码写法如下：

复制代码 
  1. bfs(s) { // s表示出发点

  2.   q = new queue()

  3.   q.push(s), visited[s] = true // 标记s为已访问

  4.   while (!q.empty()) {

  5.     u = q.pop() // 拿到当前结点 

  6.     for next in getNext(u) { // 拿到u的后继next

  7.       if (!visited[next]) { // 如果next还没有访问过 

  8.         q.push(next)

  9.         visited[next] = true

  10.       }

  11.     }

  12.   }

  13. }

使用“两段击”的代码写法如下：

复制代码 
  1. bfs(s) { // s表示出发点

  2.   cur = {s};

  3.   visited[s] = true; // 标记s为已访问

  4.   while (!cur.empty()) {

  5.     next = [];

  6.     for c in cur {

  7.       for next in getNext(c) {

  8.         if (!visited[next]) { // 如果next还没有访问过 

  9.           next.append(next);

  10.           visited[next] = true;

  11.         }

  12.       }

  13.     }

  14.     cur = next;

  15.   }

  16. }

动态规划

动态规划其实并没有太多的模板可以套，你需要通过实战练习不停地提高自己的应对能力。掌握动态规划的重点在于 6 步分析法，如下图所示：

这里我们重点看一下需要你熟练掌握的 KMP 算法模板（你还记得求 next 数组时的动态规划吗？），如下所示：

复制代码 
  1. int[] buildNext(String sub) {

  2.   final int N = sub == null ? 0 : sub.length();

  3.   int[] next = new int[N + 1];

  4.   int i = 0;

  5.   int j = -1;

  6.   next[0] = -1;

  7.   while (i < N) {

  8.     if (j == -1 || sub.charAt(i) == sub.charAt(j)) {

  9.       i++;

  10.       j++;

  11.       if (i < sub.length() && j < sub.length() &&

  12.           sub.charAt(i) == sub.charAt(j)) {

  13.         next[i] = next[j];

  14.       } else {

  15.         next[i] = j;

  16.       }

  17.     } else {

  18.       j = next[j];

  19.     }

  20.   }

  21.   return next;

  22. }

  23. int KMP(String main, String sub) {

  24.   int i = 0;

  25.   int j = 0;

  26.   final int alen = main.length();

  27.   final int blen = sub.length();

  28.   int[] next = buildNext(sub);

  29.   while (i < alen && j < blen) {

  30.     if (-1 == j || main.charAt(i) == sub.charAt(j)) {

  31.       // 如果匹配成功，那么向前走

  32.       // 这里和暴力的方法没有区别

  33.       i++;

  34.       j++;

  35.     } else {

  36.       j = next[j];

  37.     }

  38.   }

  39.   // 看一下是不是匹配完了

  40.   return j == blen ? i - blen : -1;

  41. }
