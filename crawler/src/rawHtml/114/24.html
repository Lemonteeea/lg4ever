在这个模块，我把常见的“套路”题，帮你总结成手写代码时应该准备的各种代码模板。还会把自己压箱底的独家代码模板分享给你，利用它，我多次在 10 分钟以内拿下了算法面试。

今天我先带你把数据结构部分做一个归纳和整理，方便你考前复习和平日积累。可以想象一下，如果在准备面试期间，你已经刷了很多题，那么在临近面试时还可以做些什么呢？

  • 把所有写过的代码再看一遍？

  • 把前面 20 讲的内容从头到尾再复习一遍？

  • 还是继续刷题？

在我个人看来，以上这些方法都不可取，此时最行之有效的方法是将学过的知识尽可能地压缩、再压缩，最后形成模板。整理模板，有以下几个好处。

  • 组合：其实大部分面试题都是一些算法模块的组合，并不需要我们真正去发明一个算法。

  • 速度：面试写题时速度更快，一些常用的功能性代码可以直接粘贴过去，不用在打字和调试上浪费时间。

  • 重点：可以在有限的时间里重点关注整理好的代码模板，告别“大海捞针”式的复习。

其实面试中考察的那些高频知识点，就像一块块“积木”，而面试求解过程就像“搭积木的游戏”。高效利用代码模版的技巧，能够帮助你在面试时写出更高效和 0 Bug 的代码。

说明：一些扩展知识点，我会通过练习题的形式给出来。

  34.     return ans;

  35.   }

单调栈

单调栈中经常还会用来解决这类题目：数组中元素右边第一个比元素自身小的元素的位置。

复制代码 
  1. int[] findRightSmall(int[] A) {

  2.   // 结果数组

  3.   int[] ans = new int[A.length];

  4.   // 注意，栈中的元素记录的是下标

  5.   Stack<Integer> t = new Stack();

  6.   for (int i = 0; i < A.length; i++) {

  7.     final int x = A[i];

  8.     // 每个元素都向左遍历栈中的元素完成消除动作

  9.     while (!t.empty() && A[t.peek()] > x) {

  10.       // 消除的时候，记录一下被谁消除了

  11.       ans[t.peek()] = i;

  12.       // 消除时候，值更大的需要从栈中消失

  13.       t.pop();

  14.     }

  15.     // 剩下的入栈

  16.     t.push(i);

  17.   }

  18.   // 栈中剩下的元素，由于没有人能消除他们，因此，只能将结果设置为-1。

  19.   while (!t.empty()) {

  20.     ans[t.peek()] = -1;

  21.     t.pop();

  22.   }

  23.   return ans;

  24. }

代码：Java/C++/Python

  • 数组中元素右边第一个比我大的元素的位置

  • 数组中元素左边离我最近且比我小的元素的位置

  • 数组中元素左边离我最近且比我大的元素的位置

单调栈的性质

我们将单调栈的性质总结为以下两点，更详细的介绍你可以回到《16 | 如何利用 DP 与单调队列寻找最大矩形？》进行复习。

  • 当单调递增栈中存放数组下标 i, j, k 时，其中 (i, k] 中的元素 > A[j]；

  • 当单调递增栈中存放数组下标 i, j，并且当 A[k] 入栈，会把栈顶元素 A[j]“削”出栈时，其中 (j, k) 元素 > A[j]。

我们曾经用到单调栈性质的模板代码求解最大矩形，如下所示：

复制代码 
  1. int largestRectangleArea(int[] A) {

  2.   final int N = A == null ? 0 : A.length;

  3.   // 虽然可以用Stack<Integer>，但是这里为了更快地操作，我们用

  4.   // 数组模拟栈来运行，因为我们知道最多存放的内容实际上就是N个

  5.   int top = 0;

  6.   // s[top-1]表示栈顶元素

  7.   int[] s = new int[N];

  8.   int ans = 0;

  9.   // 注意，这里我们取到了i == N

  10.   // 按理说，不应该取到i == N的。但是这时候主要是为了处理这种数组

  11.   // A = [1, 2, 3]

  12.   // 没有任何元素会出栈。

  13.   // 那么最后我们用一个0元素，把所有的元素都削出栈。

  14.   // 这样代码就可以统一处理掉。

  15.   for (int i = 0; i <= N; i++) {

  16.     // 注意：当i == N的时候，x = -1;

  17.     // 比数组中的元素都要小。

  18.     final int x = i == N ? -1 : A[i];

  19.     while (top > 0 && A[s[top - 1]] > x) {

  20.       // 计算以A[s[top]]的元素的高度的矩形。

  21.       final int height = A[s[--top]];

  22.       // i元素要将index = s[top-1]的元素出栈。

  23.       // 那么根据性质2/3：

  24.       // 此时A[s[top-1] .... i) 这个区间里面的元素都是

  25.       // 大于A[s[top-1]]的

  26.       final int rightPos = i;

  27.       // 这里需要使用性质1.

  28.       // 注意：当栈中一个元素都没有的时候，要取-1

  29.       final int leftPos = top > 0 ? s[top - 1] : -1;

  30.       final int width = rightPos - leftPos - 1;

  31.       final int area = height * width;

  32.       ans = Math.max(ans, area);

  33.     }

  34.     s[top++] = i;

  35.   }

  36.   return ans;

  37. }

代码：Java/C++/Python

循环队列

首先我们看一下使用数组来实现循环队列的写法，代码如下：

复制代码 
  1. class MyCircularQueue {

  2.     // 已经使用的元素个数

  3.     private int used = 0;

  4.     // 第一个元素所在位置

  5.     private int front = 0;

  6.     // rear是enQueue可在存放的位置

  7.     // 注意开闭原则

  8.     // [front, rear)

  9.     private int rear = 0;

  10.     // 循环队列最多可以存放的元素个数

  11.     private int capacity = 0;

  12.     // 循环队列的存储空间

  13.     private int[] a = null;

  14.     public MyCircularQueue(int k) {

  15.         // 初始化循环队列

  16.         capacity = k;

  17.         a = new int[capacity];

  18.     }

  19.     public boolean enQueue(int value) {

  20.         // 如果已经放满了

  21.         if (isFull()) {

  22.             return false;

  23.         }

  24.         // 如果没有放满，那么a[rear]用来存放新进来的元素

  25.         a[rear] = value;

  26.         // rear注意取模

  27.         rear = (rear + 1) % capacity;

  28.         // 已经使用的空间

  29.         used++;

  30.         // 存放成功!

  31.         return true;

  32.     }

  33.     public boolean deQueue() {

  34.         // 如果是一个空队列，当然不能出队

  35.         if (isEmpty()) {

  36.             return false;

  37.         }

  38.         // 第一个元素取出

  39.         int ret = a[front];

  40.         // 注意取模

  41.         front = (front + 1) % capacity;

  42.         // 已经存放的元素减减

  43.         used--;

  44.         // 取出元素成功

  45.         return true;

  46.     }

  47.     public int Front() {

  48.         // 如果为空，不能取出队首元素

  49.         if (isEmpty()) {

  50.             return -1;

  51.         }

  52.         // 取出队首元素

  53.         return a[front];

  54.     }

  55.     public int Rear() {

  56.         // 如果为空，不能取出队尾元素

  57.         if (isEmpty()) {

  58.             return -1;

  59.         }

  60.         // 注意：这里不能使用rear - 1

  61.         // 需要取模

  62.         int tail = (rear - 1 + capacity) % capacity;

  63.         return a[tail];

  64.     }

  65.     // 队列是否为空

  66.     public boolean isEmpty() {

  67.         return used == 0;

  68.     }

  69.     // 队列是否满了

  70.     public boolean isFull() {

  71.         return used == capacity;

  72.     }

  73. }

代码：Java/C++/Python

单调队列

接下来，我们看一下单调队列的实现代码。单调队列有两种，即递增队列和递减队列。由于这两种队列的代码模版非常类似，因此只需要记住其中一个就可以了，递减队列代码如下：

复制代码 
  1. class Solution {

  2.     // 单调队列使用双端队列来实现

  3.     private ArrayDeque<Integer> Q = new ArrayDeque<Integer>();

  4.     // 入队的时候，last方向入队，但是入队的时候

  5.     // 需要保证整个队列的数值是单调的

  6.     // (在这个题里面我们需要是递减的)

  7.     // 并且需要注意，这里是Q.getLast() < val

  8.     // 如果写成Q.getLast() <= val就变成了严格单调递增

  9.     private void push(int val) {

  10.         while (!Q.isEmpty() && Q.getLast() < val) {

  11.             Q.removeLast();

  12.         }

  13.         // 将元素入队

  14.         Q.addLast(val);

  15.     }

  16.     // 出队的时候，要相等的时候才会出队

  17.     private void pop(int val) {

  18.         if (!Q.isEmpty() && Q.getFirst() == val) {

  19.             Q.removeFirst();

  20.         }

  21. }

代码：Java/C++/Python

堆

由于堆往往用来实现优先级队列，因此，这里我也整理好了堆的实现的代码：

复制代码 
  1. class Heap {

  2.     private int[] a = null;

  3.     private int n = 0;

  4.     // 下沉

  5.     public void sink(int i) {

  6.         int j = 0;

  7.         int t = a[i];

  8.         // 找到i结点的左子结点

  9.         while ((j = (i << 1) + 1) < n) {

  10.             // j < n - 1判断是否有右子结点

  11.             // 如果有，并且右子结点更大，那么

  12.             // j指向右子结点

  13.             if (j < n - 1 && a[j] < a[j + 1]) {

  14.                 j++;

  15.             }

  16.             // 如果子结点比t大

  17.             // 那么t的位置还需要往后排

  18.             if (a[j] > t) {

  19.                 a[i] = a[j];

  20.                 i = j;

  21.             } else {

  22.                 // 找到了t的位置

  23.                 // 此时t是大于所有的子结点的

  24.                 break;

  25.             }

  26.         }

  27.         // 将t放在找到的位置那里

  28.         a[i] = t;

  29.     }

  30.     // 上浮

  31.     public void swim(int i) {

  32.         int t = a[i];

  33.         int par = 0;

  34.         // 如果还存在父结点

  35.         while (i > 0 && (par = (i - 1) >> 1) != i) {

  36.             // 如果父结点比t值小

  37.             if (a[par] < t) {

  38.                 a[i] = a[par];

  39.                 i = par;

  40.             } else {

  41.                 break;

  42.             }

  43.         }

  44.         a[i] = t;

  45.     }

  46.     public void push(int v) {

  47.         // push是先把元素追加到数组尾巴上，然后再执行上浮操作

  48.         a[n++] = v;

  49.         swim(n - 1);

  50.     }

  51.     public int pop() {

  52.         int ret = a[0];

  53.         a[0] = a[--n];

  54.         sink(0);

  55.         return ret;

  56.     }

  57.     public int size() {

  58.         return n;

  59.     }

  60. }

链表

要想解决链表题，我们首先需要掌几种最基本的操作，如下图所示：

不知道你是否还记得，我在《04 | 链表：如何利用“假头、新链表、双指针”解决链表题？（上）》中，将这几种操作整理成了一个代码模板。其核心思想就是链表的“第一斧”：假头。如下所示：

复制代码 
  1. 

  2. class MyLinkedList {

  3.     // 实现单链表

  4.     // 1. 假设链表中的所有节点都是 0-index的。

  5.     class ListNode {

  6.         public int val = 0;

  7.         public ListNode next = null;

  8.         public ListNode() {}

  9.         public ListNode(int x) {

  10.             val = x;

  11.         }

  12.     }

  13.     private ListNode dummy = new ListNode();

  14.     private ListNode tail = dummy;

  15.     private int length = 0;

  16.     /** Initialize your data structure here. */

  17.     public MyLinkedList() {

  18.     }

  19.     private ListNode getPreNode(int index) {

  20.         ListNode front = dummy.next;

  21.         ListNode back = dummy;

  22.         for (int i = 0; i < index; i++) {

  23.             back = front;

  24.             front = front.next;

  25.         }

  26.         return back;

  27.     }

  28.     // 获取链表中第 index 个节点的值。如果索引无效，则返回-1。

  29.     public int get(int index) {

  30.         if (index < 0 || index >= length) {

  31.             return -1;

  32.         }

  33.         return getPreNode(index).next.val;

  34.     }

  35.     // 在链表的第一个元素之前添加一个值为 val 的节点。

  36.     // 插入后，新节点将成为链表的第一个节点。

  37.     public void addAtHead(int val) {

  38.         ListNode p = new ListNode(val);

  39.         p.next = dummy.next;

  40.         dummy.next = p;

  41.         // NOTE change tail

  42.         if (tail == dummy) {

  43.             tail = p;

  44.         }

  45.         length++;

  46.     }

  47.     // 将值为 val 的节点追加到链表的最后一个元素。

  48.     public void addAtTail(int val) {

  49.         tail.next = new ListNode(val);

  50.         tail = tail.next;

  51.         length++;

  52.     }

  53.     // 在链表中的第 index 个节点之前添加值为 val  的节点。

  54.     // 1. 如果 index 等于链表的长度，则该节点将附加到链表的末尾。

  55.     // 2. 如果 index 大于链表长度，则不会插入节点。

  56.     // 3. 如果index小于0，则在头部插入节点。

  57.     public void addAtIndex(int index, int val) {

  58.         if (index > length) {

  59.             return;

  60.         } else if (index == length) {

  61.             addAtTail(val);

  62.             return;

  63.         } else if (index <= 0) {

  64.             addAtHead(val);

  65.             return;

  66.         }

  67.         ListNode pre = getPreNode(index);

  68.         ListNode p = new ListNode(val);

  69.         p.next = pre.next;

  70.         pre.next = p;

  71.         // NOTE: here tail has been changed

  72.         length++;

  73.     }

  74.     // 如果索引 index 有效，则删除链表中的第 index 个节点。

  75.     public void deleteAtIndex(int index) {

  76.         if (index < 0 || index >= length) {

  77.             return;

  78.         }

  79.         ListNode pre = getPreNode(index);

  80.         // NOTE: delete -> change tail

  81.         if (tail == pre.next) {

  82.             tail = pre;

  83.         }

  84.         length--;

  85.         pre.next = pre.next.next;

  86.     }

  87. }

使用栈来实现的前序遍历的代码如下（解析在注释里）：

复制代码 
  1. class Solution {

  2.     public List<Integer> preorderTraversal(TreeNode root) {

  3.         // 用来进行递归的栈

  4.         Stack<TreeNode> s = new Stack<>();

  5.         // 用来存放遍历的结果，不算在空间复杂度里面 

  6.         List<Integer> ans = new ArrayList<>();

  7.         // 开始利用栈来进行遍历

  8.         while (root != null || !s.empty()) {

  9.             // 模拟递归的压栈过程

  10.             while (root != null) {

  11.                 s.push(root);

  12.                 ans.add(root.val);

  13.                 root = root.left;

  14.             }

  15.             // 当无法压栈的时候，将root.right进行压栈

  16.             root = s.peek();

  17.             s.pop();

  18.             root = root.right;

  19.         }

  20.         return ans;

  21.     }

  22. }

代码：Java/C++/Python

复制代码 
  1. class Solution {

  2.     public List<Integer> inorderTraversal(TreeNode root) {

  3.         Stack<TreeNode> s = new Stack<>();

  4.         List<Integer> ans = new ArrayList<>();

  5.         // 注意这里的判断条件，需要root 或 stack非空

  6.         while (root != null || !s.empty()) {

  7.             // 往左边走，连续入栈，直到不能再走为止

  8.             while (root != null) {

  9.                 s.push(root);

  10.                 root = root.left;

  11.             }

  12.             // 到达了最左边，把结点弹出来，进行遍历

  13.             root = s.peek();

  14.             s.pop();

  15.             ans.add(root.val);

  16.             // 转向右子树

  17.             root = root.right;

  18.         }

  19.         // 返回遍历的结果

  20.         return ans;

  21.     }

  22. }

后序遍历

采用递归实现的后序遍历代码模板如下（解析在注释里）：

复制代码 
  1. void postOrder(TreeNode root, List<Integer> ans) {

  2.   if (root != null) {

  3.     // 先遍历左子树

  4.     postOrder(root.left, ans);

  5.     // 最后遍历右子树

  6.     postOrder(root.right, ans);

  7.     // 然后遍历中间的根结点

代码：Java/C++/Python

采用非递归的后序遍历代码如下（解析在注释里）：

复制代码 
  1. class Solution {

  2.     public List<Integer> postorderTraversal(TreeNode t) {

  3.         // 存放遍历的结果

  4.         List<Integer> ans = new ArrayList<>();

  5.         // pre表示遍历时前面一个已经遍历过的结点

  6.         TreeNode pre = null;

  7.         Stack<TreeNode> s = new Stack<>();

  8.         // 如果栈中还有元素，或者当前结点t非空

  9.         while (!s.isEmpty() || t != null) {

  10.             // 顺着左子树走，并且将所有的元素压入栈中

  11.             while (t != null) {

  12.                 s.push(t);

  13.                 t = t.left;

  14.             }

  15.             // 当没有任何元素可以压栈的时候

  16.             // 拿栈顶元素，注意这里并不将栈顶元素弹出

  17.             // 因为在迭代时，根结点需要遍历两次，这里需要判断一下

  18.             // 右子树是否遍历完毕

  19.             t = s.peek();

  20.             // 如果要遍历当前结点，需要确保右子树已经遍历完毕

  21.             // 1. 如果当前结点右子树为空，那么右子树没有遍历的必要

  22.             // 需要将当前结点放到ans中

  23.             // 2. 当t.right == pre时，说明右子树已经被打印过了

  24.             // 那么此时需要将当前结点放到ans中

  25.             if (t.right == null || t.right == pre) {

  26.                 // 右子树已经遍历完毕，放到ans中。

  27.                 ans.add(t.val);

  28.                 // 弹栈

  29.                 s.pop();

  30.                 // 因为已经遍历了当前结点，所以需要更新pre结点

  31.                 pre = t;

  32.                 // 已经打印完毕。需要设置为空，否则下一轮循环

  33.                 // 还会遍历t的左子树。

  34.                 t = null;

  35.             } else {

  36.                 // 第一次走到t结点，不能放到ans中，因为t的右子树还没有遍历。

  37.                 // 需要将t结点的右子树遍历

  38.                 t = t.right;

  39.             }

  40.         }

  41.         return ans;

  42.     }

  43. }

代码：Java/C++/Python

【面试建议】在面试的时候，大部分情况都应该优先写递归的代码，除非面试官特别要求你必须使用“非递归”来实现。主要有以下几点原因：

  • 递归代码更加简单，因此不容易出错；

  • 不要为了“炫技”展示“非递归”代码；

  • 如果我们要进行二叉树上的搜索、DP、二分等情况的时候，“非递归”的代码往往会增加代码的复杂度，面试的时候不容易完全写对。

在这一讲中，我们通过整理代码模板，将“第 01 讲” 到“第 07 讲”学习的所有知识点都整理好了。这样你复习起来是不是压力要小很多呢。下面我再和你分享两个代码模板的“小秘密”。

模板代码要精练

其实在整理模板的时候，要尽量将代码压缩得越短越好（指的并不是不换行），代码压缩得短，有以下好处：

  • 如果是自己熟悉的代码，在需要记忆的情况下，越短越好记；

  • 较短的代码可以更精练，一眼看上去没有那么大的心理压力。

比如就我自己而言，在复习并查集的代码时，就经常使用下面这段更短的代码：

复制代码 
  1. int Find(int x) { return x == F[x] ? x : F[x] = Find(F[x]); }

  2. void Union(int x, int y) {  F[find(x)] = find(y); }

自己整理可复用的代码模版

和你分享一下自己整理的模板的好处。主要是基于以下两点原因。

1. 变量的命名要有规律，而这些规律都是自己平时约定使用的，当你复习代码时会更熟练，比如：

1）返回值一律设置为 ans 或者 ret；

2）遍历下标设置为 i,j,k；

3）长度变量设置为 len。

2. 同一个算法往往有很多种写法，自己的写法会更熟悉，而且可以不断迭代和复用。

所以，本讲的练习题，就是希望你能把“第 01 讲”到“第 07 讲”刷过的题的代码整理成模板。当临近面试，你只需要对着思维导图和代码模板过一下思路就可以了。
