在前面第 28 讲“设计性能守卫系统：完善 CI/CD 流程”中我们提到了 Puppeteer。事实上，以 Puppeteer 为代表的 Headless 浏览器在 Node.js 中的应用极为广泛，这一讲，就让我们对 Puppeteer 进行深入分析和应用。

Puppeteer 介绍和原理

我们先对 Puppeteer 进行一个基本介绍。（Puppeteer 官方地址）

Puppeteer 是一个 Node 库，它提供了一整套高级 API，通过 DevTools 协议控制 Chromium 或 Chrome。正如其翻译为“操纵木偶的人”一样，你可以通过 Puppeteer 提供的 API 直接控制 Chrome，模拟大部分用户操作，进行 UI 测试或者作为爬虫访问页面来收集数据。

整个定义非常好理解，这里需要开发者注意的是，Puppeteer 在 1.7.0 版本之后，会同时给开发者提供：

  • Puppeteer

  • Puppeteer-core

两个版本。它们的区别在于载入安装 Puppeteer 时，是否会下载 Chromium。Puppeteer-core 默认不下载 Chromium，同时会忽略所有 puppeteer_* 环境变量。对于开发者来说，使用 Puppeteer-core 无疑更加轻便，但是需要提前保证环境中已经具有可执行的 Chromium（具体说明可见puppeteer vs puppeteer-core）。

具体 Puppeteer 的应用场景有：

  • 为网页生成页面 PDF 或者截取图片；

  • 抓取 SPA（单页应用）并生成预渲染内容；

  • 自动提交表单，进行 UI 测试、键盘输入等；

  • 创建一个随时更新的自动化测试环境，使用最新的 JavaScript 和浏览器功能直接在最新版本的 Chrome 中执行测试；

  • 捕获网站的timeline trace，用来帮助分析性能问题；

  • 测试浏览器扩展。

下面我们就梳理一些 Puppeteer 应用的重点场景，并详细介绍如何使用 Puppeteer 实现一个高性能的海报 Node.js 服务。

Puppeteer 在 SSR 中的应用

区别于第 27 讲介绍的“同构渲染架构：实现一个 SSR 应用”，使用 Puppeteer 实现服务端预渲染出发点完全不同。这种方案最大的好处是不需要对项目代码进行任何调整，却能获取到 SSR 应用的收益。当然，相比同构渲染，基于 Puppeteer 技术的 SSR 在灵活性和扩展性上都有所局限。甚至在 Node.js 端渲染的性能成本也较高，不过该技术也逐渐落地，并在很多场景发挥了重要价值。

该页面是一个典型的 CSR 页面，依靠 Ajax，实现了页面动态化渲染。

当在 Node.js 端使用 Puppeteer 渲染时，我们可以实现ssr.mjs，完成渲染任务，如下代码：

复制代码 
  1. import puppeteer from 'puppeteer';

  2. // 将已经渲染过的页面，缓存在内存中

  3. const RENDER_CACHE = new Map();

  4. async function ssr(url) {

  5. 	// 命中缓存

  6.   if (RENDER_CACHE.has(url)) {

  7.     return {html: RENDER_CACHE.get(url), ttRenderMs: 0};

  8.   }

  9.   const start = Date.now();

  10.   // 使用 Puppeteer launch 一个无头浏览器

  11.   const browser = await puppeteer.launch();

  12.   const page = await browser.newPage();

  13.   try {

  14.     // 访问页面地址直到页面网络状态为 idle

  15.     await page.goto(url, {waitUntil: 'networkidle0'});

  16.     // 确保 #posts 节点已经存在

  17.     await page.waitForSelector('#posts');

  18.   } catch (err) {

  19.     console.error(err);

  20.     throw new Error('page.goto/waitForSelector timed out.');

  21.   }

  22. 	// 获取 html 内容

  23.   const html = await page.content(); 

  24.   // 关闭无头浏览器

  25.   await browser.close();

  26.   const ttRenderMs = Date.now() - start;

  27.   console.info(`Headless rendered page in: ${ttRenderMs}ms`);

  28. 	// 进行缓存存储

  29.   RENDER_CACHE.set(url, html);

  30.   return {html, ttRenderMs};

  31. }

  32. export {ssr as default};

对应server.mjs代码：

复制代码 
  1. import express from 'express';

  2. import ssr from './ssr.mjs';

  3. const app = express();

  4. app.get('/', async (req, res, next) => {

  5.   // 调用 SSR 方法渲染页面

  6.   const {html, ttRenderMs} = await ssr(`xxx/index.html`);

  7.   res.set('Server-Timing', `Prerender;dur=${ttRenderMs};desc="Headless render time (ms)"`);

  8.   return res.status(200).send(html);

  9. });

  10. app.listen(8080, () => console.log('Server started. Press Ctrl+C to quit'));

当然上述实现比较简陋，只是进行原理说明。如果更进一步，我们可以从以下几个角度进行优化：

  • 改造浏览器端代码，防止重复请求接口；

  • 在 Node.js 端，abort 掉不必要的请求，以得到更快的服务端渲染响应速度；

  • 将关键资源内连进 HTML；

  • 自动压缩静态资源；

  • 在 Node.js 端，渲染页面时，重复利用 Chrome 实例。

这里我们用简单代码进行说明：

复制代码 
  1.   import express from 'express';

  2.   import puppeteer from 'puppeteer';

  3.   import ssr from './ssr.mjs';

  4.   // 重复使用 Chrome 实例

  5.   let browserWSEndpoint = null;

  6.   const app = express();

  7. 

  8.   app.get('/', async (req, res, next) => {

  9.     if (!browserWSEndpoint) {

  10.       // 一下两行代码不必随着渲染重复执行

  11.       const browser = await puppeteer.launch();

  12.       browserWSEndpoint = await browser.wsEndpoint();

  13.     }

  14. 

  15.     const url = `${req.protocol}://${req.get('host')}/index.html`;

  16.     const {html} = await ssr(url, browserWSEndpoint);

  17. 

  18.     return res.status(200).send(html);

  19.   });

Puppeteer 结合 Lighthouse 应用场景

在第 28 讲“设计性能守卫系统：完善 CI/CD 流程”中我们也提到了 Lighthouse，既然 Puppeteer 可以和 Jest 结合实现一个端到端测试框架，当然也可以和 Lighthouse 结合——这就是一个简单的性能守卫系统的雏形。

  10.     // 默认配置

  14.         disableDeviceEmulation: true,

  15.         defaultViewport: {

  16.             width: 1200,

  18.         },

  42.     fs.writeFile('report.html', html, (err) => {

  44.             console.error(err);

这里需要特别强调的是，为了实现最好的性能，我们设计了一个链接池来存储 Puppeteer 实例，以备所需，如下图所示：

在实现上，我们依赖generic-pool库，这个库提供了 Promise 风格的通用池，可以用来对一些高消耗、高成本资源的调用实现防抖或拒绝服务能力，一个典型场景是对数据库的连接。这里我们把它用于 Puppeteer 实例的创建，如下代码所示：

复制代码 
  1. const puppeteer = require('puppeteer')

  2. const genericPool = require('generic-pool')

  3. const createPuppeteerPool = ({

  4.   // pool 的最大容量

  5.   max = 10,

  6.   // pool 的最小容量

  7.   min = 2,

  8.   // 连接在池中保持空闲而不被回收的最小时间值

  9.   idleTimeoutMillis = 30000,

  10.   // 最大使用数

  11.   maxUses = 50,

  12.   // 在连接池交付实例前是否先经过 factory.validate 测试

  13.   testOnBorrow = true,

  14.   puppeteerArgs = {},

  15.   validator = () => Promise.resolve(true),

  16.   ...otherConfig

  17. } = {}) => {

  18.   const factory = {

  19.   	// 创建实例

  20.     create: () =>

  21.       puppeteer.launch(puppeteerArgs).then(instance => {

  22.         instance.useCount = 0

  23.         return instance

  24.       }),

  25.     // 销毁实例

  26.     destroy: instance => {

  27.       instance.close()

  28.     },

  29.     // 验证实例可用性

  30.     validate: instance => {

  31.       return validator(instance).then(valid =>

  32.         // maxUses 小于 0 或者 instance 使用计数小于 maxUses 时可用

  33.         Promise.resolve(valid && (maxUses <= 0 || instance.useCount < maxUses))

  34.       )

  35.     }

  36.   }

  37.   const config = {

  38.     max,

  39.     min,

  40.     idleTimeoutMillis,

  41.     testOnBorrow,

  42.     ...otherConfig

  43.   }

  44.   // 创建连接池

  45.   const pool = genericPool.createPool(factory, config)

  46.   const genericAcquire = pool.acquire.bind(pool)

  47.   // 池中资源连接时进行的操作

  48.   pool.acquire = () =>

  49.     genericAcquire().then(instance => {

  50.       instance.useCount += 1

  51.       return instance

  52.     })

  53.   pool.use = fn => {

  54.     let resource

  55.     return pool

  56.       .acquire()

  57.       .then(r => {

  58.         resource = r

  59.         return r

  60.       })

  61.       .then(fn)

  62.       .then(

  63.         result => {

  64.           // 释放资源

  65.           pool.release(resource)

  66.           return result

  67.         },

  68.         err => {

  69.           pool.release(resource)

  70.           throw err

  71.         }

  72.       )

  73.   }

  74.   return pool

  75. }

  76. module.exports = createPuppeteerPool

有了“武器弹药”，我们来看看渲染一个页面为海报的具体逻辑。如下代码所示render方法，该方法支持接受一个 URL 也支持接受具体的 HTML 字符串去生成相应海报：

复制代码 
  1. // 获取连接池

  2. const pool = require('./pool')

  3. const config = require('./config')

  4. const render = (ctx, handleFetchPicoImageError) =>

  5.   // 使用连接池资源

  6.   pool.use(async browser => {

  7.     const { body, query } = ctx.request

  8.     // 打开新的页面

  9.     const page = await browser.newPage()

  10.     // 服务支持直接传递 HTML 字符串内容

  11.     let html = body

  12. 		// 从请求服务的 query 获取默认参数

  13.     const {

  14.       width = 300,

  15.       height = 480,

  16.       ratio: deviceScaleFactor = 2,

  17.       type = 'png',

  18.       filename = 'poster',

  19.       waitUntil = 'domcontentloaded',

  20.       quality = 100,

  21.       omitBackground,

  22.       fullPage,

  23.       url,

  24.       useCache = 'true',

  25.       usePicoAutoJPG = 'true'

  26.     } = query

  27.     let image

  28.     try {

  29.     	// 设置浏览器视口

  30.       await page.setViewport({

  31.         width: Number(width),

  32.         height: Number(height),

  33.         deviceScaleFactor: Number(deviceScaleFactor)

  34.       })

  35.       if (html.length > 1.25e6) {

  36.         throw new Error('image size out of limits, at most 1 MB')

  37.       }

  38. 			// 访问 URL 页面

  39.       await page.goto(url || `data:text/html,${html}`, {

  40.         waitUntil: waitUntil.split(',')

  41.       })

  42. 			// 进行截图

  43.       image = await page.screenshot({

  44.         type: type === 'jpg' ? 'jpeg' : type,

  45.         quality: type === 'png' ? undefined : Number(quality),

  46.         omitBackground: omitBackground === 'true',

  47.         fullPage: fullPage === 'true'

  48.       })

  49.     } catch (error) {

  50.       throw error

  51.     }

  52.     ctx.set('Content-Type', `image/${type}`)

  53.     ctx.set('Content-Disposition', `inline; filename=${filename}.${type}`)

  54.     await page.close()

  55.     return image

  56.   })

  57. module.exports = render

至此，基于 Puppeteer 的海报系统就已经开发完成了。它是一个对外的 Node.js 服务。
